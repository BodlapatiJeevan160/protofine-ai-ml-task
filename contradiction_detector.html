How it works

Five independent rule layers. Each fires on different contradiction patterns and contributes points to a 0–100 severity score. No LLM, no API, fully deterministic.

Five detection layers
1

Antonym-pair scanner

Checks every sentence pair for 40+ known opposite-meaning phrases: stateless/stateful, never writes/writes to disk, safe indefinitely/should not. Fires when sentence i contains term A and sentence j contains its antonym B.

+40 pts
2

Absolute-negation flip

Detects categorical claims using never, always, completely, no X. Fires when another sentence contradicts that absolute on the same shared subject noun.

+35 pts
3

Quantity contradiction

Catches numeric flips: zero/none/nothing in one sentence versus a non-zero quantity in another about the same subject. Also catches all/every versus some/sometimes/partial.

+30 pts
4

Scope-violation detector

Catches when text claims universal scope (in all cases, without exception, guaranteed) then later narrows it (sometimes, under certain conditions) on the same subject.

+25 pts
5

Shared-subject negation

Compares sentence pairs sharing ≥2 key nouns. If one sentence asserts and the other negates (using not/no/without/cannot), flags as candidate contradiction — only if not already covered by layers 1–4.

+20 pts

Prompt guide

How to write text that gives the detector the best chance of finding real issues — and what it cannot catch.

What counts as a contradiction?

Real contradiction

"Redis never writes data to disk." … "Redis uses AOF logs to write all data to disk." — mutually exclusive. If one is true, the other must be false.

Not a contradiction

"Python is slow for math, but NumPy is fast." — Both can be simultaneously true. That's nuance, not a logical conflict. The detector won't flag this.

Tips for best results

1Submit complete answers, not fragments. Contradictions often span an intro claim and a later exception — the full context is needed.
23–8 sentences works best. Too short means no pairs to compare. Too long can dilute the signal.
3Technical and factual content. The engine targets logical claims — code explanations, medical advice, policy docs. Not poetry.
4Don't test with metaphors. "Both a blessing and a curse" is figurative language, not a logical contradiction. The engine won't flag it.
5Low confidence = human review. A score under 35 means borderline — don't auto-reject, check manually.

What this does not catch

This tool checks logic, not facts. It will not catch fabricated statistics, made-up citations, outdated information, or vague unhelpful answers. Use it as one layer of a multi-layer trust system alongside a fact-checker and citation verifier.
